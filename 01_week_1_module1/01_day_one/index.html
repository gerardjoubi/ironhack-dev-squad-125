Wrap up

HTML Stands for Hyper Text Markup Langage.
This langage provides you with the structure of your apps.
It's pretty straight forward but don't underestimate all the problems a poorly structured HTML code can lead you to !!!
To check if your code is valid, make good use of the w3c validator tool (online ou via a plugin in your favorite text-editor).
The complete list of HTML tags can be found on MDN.
REMEMBER : each HTML document is a tree of nodes.
Each node can be either a tag OR a text. The tree is organised as a collection of parent/children and sibling(s) nodes.

CSS Stands for Cascading Stylesheet.
This langage is used to provide you with the style of your apps.
One of the hardest things to master about CSS are selectors. So please make sure to wax'on wax'off the CSS DINER game.
This way, you will be able to target easily whatever you want on a web page.
Plus it's going to be super usefull when we will be dealing with JS and The D.O.M

To sum up on CSS Positioning : flex is the easiest way to go when displaying content. It's powerfull and responsive-design oriented.
So don't forget to practice whith the game flexbox froggy when you have time.
We'll see one other technology later, Grid Layouts, wich combined with flexbox provide you with all you need to produce complex layouts.
